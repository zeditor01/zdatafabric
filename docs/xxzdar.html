Z Data at REST

The ongoing costs of making low-latency copies of z/OS data for every new application that needs current operational data access are HUGE, not to mention the layers of technical debt that can be added to your systems to be maintained forwever. Some use cases may genuinely have some need for a physical copy of the mainframe data, but many use cases can be served better and cheaper by just deploying an API to serve the data on demand to any application on the internet

It is true that the predominant pattern for serving new cloud applications with current operational data is to stream the data off the mainframe to a digital integration hub of some sort. Many sites are so familiar with that pattern that they don't fully evaluate the merits of serving APIs directly from the original data source.

The articles in this section contain worked examples of how to use various IBM software solutions to serve data and information APIs directly from z/OS.

IBM's Z Data at REST Options

IBM's premier solution for serving REST APIs from mainframe systems is z/OS Connect Enterprise Edition. It is designed to act as an API gateway into z/OS transactions as well as z/OS data.

This sets of papers is focussed on the details of how the underlying data services are created and implemented. They also show how APIs can be published (whether by z/OS Connect, or API Connect) based on those data services.

DB2 DDF is a standard component of Db2 z/OS.

  • It can serve REST APIs based on calls to SQL and Stored Procedures.
  • It can serve a variety of SQL-based APIs ( JDBC, ODBC etc... ).
  • It is tightly integrated into the Db2 z/OS service (Reliability, Performance, Secuirty etc...)
