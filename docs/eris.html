3 Patterns for a Data Fabric for Mainframe Data

  • Z Data in Motion
  • Z Data at REST
  • ERIS

ERIS (Environment for Realtime Information Services) uses a combination of tools and techniques from both "Z Data in Motion" amd "Z Data at REST" patterns to provide a framework for providing realtime data APIs for multiple purposes, by utilising the industry-leading IBM Z hardware platform, with a combination of z/OS systems of record, and containerised data and analtical solutions on Linux.

The ERIS approach is an enabler for the Hybrid Cloud solutions that most mainframe-using enterprises seek. They want to move large portions of their business to the public cloud, but need tight realtime integration with the systems of record that will continue to run on mainframe for many years to come. By deploying the now-ubiquitous Digital Integration Hub building blocks on a containerised Linux for Z platform you get a free ride on all the high availability and disaster recovery infrastructure that already exists for the z/OS systems of record, and you get a more secure environment for all the open source and containerised solutions used for providing digital integration APIs.

Cloudpak for Data.

  • Is served by Db2 integrated synchronisation for access to current Db2 data in transactions or analytical formats.
  • Supports multiple optional services spannng data governance and AI.
  • Is deployable on any Openshift platform: Cloud, Midrange, Linux for Z etc...

Cloudpak for Integration

  • Accepts feeds from non-DB2 data.
  • Supports curation of data for services.
  • Publishes and Manages APIs on curated data

Z Data Mesh.

  • All APIs from raw Z data, or curated data, managed inside a secure Z hardware platform (ErisONE).
  • Platform for Data Owners to manage how data is published for consumption.
